Command-line interface usage#

Main command#

qdeploy-toolbelt#

Main command.

Args:

cli_context (click.Context): Click context scenario_filepath (Path): path to a scenario file to use clear (bool): option to clear the terminal berfore any other step verbose (bool): option to force the verbose mode

Example
qgis-deployment-toolbelt -c --verbose check

qdeploy-toolbelt [OPTIONS] COMMAND1 [ARGS]... [COMMAND2 [ARGS]...]...

Options

-c, --clear#

Clear the terminal before the execution.

Default

False

-v, --verbose#

Set output verbosity to the maximum level, overriding the configuration option.

Default

False

-s, --scenario <scenario_filepath>#

Scenario file to use.

Default

scenario.qdt.yml

--version#

Display CLI version

Sub-commands#

check#

Perform checks about CLI requirements.

Args:

cli_context (click.Context): Click context

check [OPTIONS]

clean#

Delete logs and output folders older than the frequency set in the configuration: CLEAN_FREQUENCY.

param click.core.Context cli_context

Click context

param int days_older

number of days after which the file is deleted.

param bool remove_empty_folders

if passed, empty folders will be deleted too

clean [OPTIONS]

Options

-d, --days-older <days_older>#

Number of days after which the file is deleted.

Default

30

-e, --remove-empty-folders#

Remove empty folders.

Default

True

Environment variables

QGIS_DEPLOYMENT_CLEAN_DAYS_OLDER

Provide a default for -d
